# 图形化 EMS 逻辑配置实现计划

> **面向 AI 代理的工作者：** 必需子技能：使用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实现此计划。步骤使用复选框（`- [ ]`）语法来跟踪进度。

**目标：** 在保留 `legacyEms` 的基础上，新增可由平台图形化生成 JSON、边端解释执行的 `graphEms` EMS 策略图能力。

**架构：** 平台端生成 `shuntong_ems_graph.json`，边端 `ComputeEngineService` 通过 `script.type=graphEms` 加载并执行。`GraphEmsEngine` 复用 `PointStoreRouter` 读写共享内存和提交 `PendingWriteCommand`，首阶段用领域节点复刻舜通 EMS 逻辑。

**技术栈：** C++17、现有 JSON 解析器、`PointStoreRouter`、`MemoryPointStore`、`ComputeEngineService`、现有 `legacy_ems_test`。
