# 生产上线检查清单

本文用于边端网关正式投产前的最后确认。默认安装目录为 `/opt/modbus-gateway`。

## 1. 身份与配置

- `config/runtime/device_identity.json` 中的 `machineCode` 必须是平台登记的唯一网关编号。
- `imei`、`serialNumber`、`model`、`firmwareVersion` 应写入实际设备信息。
- 串口/协议设备 JSON 不再配置 `machineCode`，只配置 `meterCode`、协议、点表、共享内存和本地存储。
- `mqtt-service.json` 和 `monitor-service.json` 的 `deviceConfigFiles[]` 只引用需要运行的设备文件；未引用的设备 JSON 不会启动。
- 本机 DI/DO 如果参与业务，必须引用 `device_dio.json`，并确认 `local_dio` 点位的 GPIO 映射和低有效配置。

## 2. 服务与共享内存

投产前执行：

```sh
/opt/modbus-gateway/bin/gateway-run.sh list
/opt/modbus-gateway/bin/gateway-run.sh health
/opt/modbus-gateway/bin/gateway-run.sh smoke
/opt/modbus-gateway/bin/gateway-run.sh stats
```

验收标准：

- `list` 只出现现场实际需要运行的协议驱动、`event-engine@mqtt-service`、`mqtt-driver@mqtt-service`、`system-monitor@monitor-service`；如果启用本机 DIDO，应包含 `dio-driver@device_dio.service`。
- `health` 中所有目标服务为 `active`。
- `stats` 能立即返回，不允许卡住；`latest`、`pendingWrites`、`pointUpdates` 不应接近容量上限。
- `smoke` 最终必须输出 `fail=0`；`warn` 项需要人工确认并记录原因。
- 普通 `systemctl restart gateway-services.service` 应在数秒到十几秒内完成。采集驱动已使用可中断睡眠，正常停止时日志应出现 `gateway daemon stopped`。

本机 DIDO 投产前需要额外确认：

- `DI1-DI18` 低有效，短接或输入有效时平台值应为 `1`。
- `DO1-DO8` 低有效，平台写 `1` 时继电器闭合，写 `0` 时继电器断开。
- `pointctl write --index 420001 --value 1` 能让 DO1 闭合，`--value 0` 能断开。
- 不要使用 `rgpio.sh` 验证 DO 输出状态；该脚本会把 GPIO 改为输入模式，影响 DO 测试。

如果 `pointctl stats` 报 `shared memory mutex lock timed out`，按维护流程清理：

```sh
systemctl stop gateway-services.service
/opt/modbus-gateway/bin/gateway-services.sh stop
rm -f /dev/shm/gateway_point_store*
systemctl start gateway-services.service
```

不要在服务运行中删除共享内存。

## 3. MQTT 与 TLS

- 内置 MQTT Publisher 支持 TCP、TLS、用户名密码、MQTT3/MQTT5。
- 跨公网或不可信网络必须启用 TLS，推荐直接配置 `broker=ssl://...:8883` 和 `tls.enabled=true`。
- 边端需要具备 OpenSSL 运行库；如果系统无法提供 OpenSSL，可使用 `mqtt-tls-tunnel@.service` 本机 stunnel 兜底。
- 下行命令、OTA、系统监测、配置拉取必须发布到 `<baseTopic>/<machineCode>`。

投产前执行：

```sh
ldconfig -p | grep -E 'libssl|libcrypto'
grep -nE '"broker"|"tls"' /opt/modbus-gateway/config/runtime/apps/mqtt-service.json
grep -nE '"broker"|"tls"' /opt/modbus-gateway/config/runtime/apps/monitor-service.json
```

验收标准：

- TLS 配置应使用 `ssl://` / `tls://` / `mqtts://` 或 `tls.enabled=true`。
- `tls.caFile` 指向的 CA 文件必须存在；如果为空，必须确认系统 CA 路径完整。
- `tls.certFile` 和 `tls.keyFile` 要么都为空，要么成对存在。
- 生产禁止配置 `tls.insecureSkipVerify=true`。
- 如果使用本机 `mqtt-tls-tunnel@.service` 兜底，app broker 必须指向本机 stunnel 监听端口，且 `config/runtime/tls/*-stunnel.conf` 的 `connect`、`checkHost`、`CAfile` 必须替换为真实生产值，不能保留 `your-mqtts-broker.example.com`。

## 4. OTA 与回滚

- `ota.retentionCount` 保持 `3`，同一网关只保留最近 3 个包。
- `ota-apply.sh` 默认保留现有 `device_identity.json`，除非 manifest 显式声明 `updateIdentity=true`。
- OTA 配置包应携带 `gateway-services.sh`、systemd 模板、`ota-apply.sh`、`ota-rollback.sh`。
- 升级后检查 `edge/ota/status/<machineCode>` 和 `/opt/modbus-gateway/ota/staging/upgrade_history.log`。

## 5. 断链缓存与磁盘

- 实时数据使用预分配 ring 文件，默认 1GB。
- 告警、变位、OTA 状态使用 SQLite outbox，默认按自然月保留 12 个月。
- 投产前确认根分区余量：

```sh
df -h /opt/modbus-gateway
```

## 6. 现场故障边界

- `modbus response too short` 属于现场设备响应异常，不应导致进程退出。
- `point.index already claimed by another active process` 表示有重复点位或旧进程未释放租约；先查进程和 `gateway-run.sh list`，确认后再停服清理共享内存。
- `shared pending write queue is full` 表示下发写命令过快或设备执行过慢，需要限流或增大队列容量后全量重启。
