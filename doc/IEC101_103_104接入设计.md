# IEC101/103/104 接入设计

边端新增独立 `IecDriver`，不挂载到 Modbus、DLT645 或 CAN 驱动。采集结果写入现有 `MemoryPointStore`，由 `MqttDriver` 统一负责全量、实时和补发，因此采集驱动内不直接推 MQTT。

## 支持范围

- IEC104：TCP 链路，支持 STARTDT、总召唤和常见遥信/遥测 ASDU 解码。
- IEC101：串口 FT1.2 链路，支持总召唤和常见遥信/遥测 ASDU 解码。
- IEC103：串口 FT1.2 链路，按 `FUN/INF` 点位映射解析常见状态和测量信息。
- IEC103 TCP：TCP 上承载 FT1.2 变长帧，点位映射与 IEC103 串口一致。

当前第一版侧重采集闭环。遥控/设点写入默认返回不支持，后续需要按现场设备的单点/双点、选择执行和确认流程单独启用。

## 配置

协议级字段放在 `protocol.iec`：

| 字段 | 说明 |
| --- | --- |
| `transportMode` | `serial` 或 `tcp`，IEC103 两种都支持 |
| `commonAddress` | 公共地址 |
| `cotSize` | 传送原因字节数，IEC104/101 常用 2 |
| `caSize` | 公共地址字节数，IEC104/101 常用 2 |
| `ioaSize` | 信息体地址字节数，IEC104/101 常用 3 |
| `linkAddress` | IEC101/103 链路地址 |
| `linkAddressSize` | IEC101/103 链路地址字节数 |
| `interrogationQualifier` | 总召唤限定词，默认 20 |
| `pollTimeoutMs` | 每次轮询接收窗口 |
| `maxPollFrames` | 单次轮询最多处理帧数 |
| `pollOnCollect` | 是否每次采集周期主动发送总召唤 |

点位字段放在 `read.iec`：

| 字段 | 说明 |
| --- | --- |
| `ioa` | IEC101/104 信息体地址；未填时可复用点位 `address` |
| `typeId` | ASDU 类型标识，常用 1/3/9/11/13/15 |
| `commonAddress` | 点位所在公共地址，可选 |
| `cause` | 传送原因过滤，可选 |
| `functionType` | IEC103 FUN |
| `informationNumber` | IEC103 INF |
| `valueKind` | 可选语义类型 |

单设备配置可以直接使用顶层 `points`，同时保留 `meters: []`；多逻辑设备配置继续使用 `meters[].points`。

示例文件：

- `config/examples/device_iec104_example.json`
- `config/examples/device_iec101_example.json`
- `config/examples/device_iec103_serial_example.json`
- `config/examples/device_iec103_tcp_example.json`

## 部署

`gateway-services.sh` 会扫描运行时设备配置：

- `protocol.type=iec104`
- `protocol.type=iec101`
- `protocol.type=iec103`
- `protocol.type=iec103_tcp`
- `protocol.type=iec103_serial`

只有被 `mqtt-service.json` 或 `monitor-service.json` 的 `deviceConfigFiles[]` 引用且 `enabled=true` 的 IEC 配置会启动 `iec-driver@<配置文件名>.service`。
