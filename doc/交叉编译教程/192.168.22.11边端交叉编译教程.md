# 192.168.22.11 边端交叉编译教程

## 适用范围

本文档统一记录当前边端项目在 `192.168.22.11` 交叉编译机上的构建方式，包括：

- 边端 C++ 驱动和服务程序。
- KY-EMS Qt 本地界面程序。
- 出厂初始化包中的二进制同步要求。

当前目标平台是全志 aarch64，不要使用 RK3568 工具链、RK3568 sysroot 或 Windows MinGW Qt 路径。

## 编译机路径

- 边端源码：`/mnt/hgfs/Embedded/Gateway-zk`
- KY-EMS 源码：`/mnt/hgfs/Embedded/KY-EMS`
- 边端工具链：
  `/home/tronlong/Linux/SZR/aarch64/gcc-linaro-6.3.1-2017.05-x86_64_aarch64-linux-gnu`
- CMake 工具链文件：`toolchains/aarch64-linux-gnu.cmake`
- KY-EMS 目标 sysroot：`/opt/ky-cross/allwinner-sysroot`
- KY-EMS 主机 Qt 工具：`/opt/Qt5.12.9/5.12.9/gcc_64/bin`

交叉编译工具链不一定在登录 shell 的 `PATH` 中，项目脚本会自动补齐所需路径。

## 边端驱动编译

在 `192.168.22.11` 上执行：

```sh
cd /mnt/hgfs/Embedded/Gateway-zk
sh tools/build_edge_aarch64.sh --clean
```

该命令默认构建到 `build-aarch64-cross`，不会覆盖现有 `build-aarch64` 产物。

如果需要将已验证产物发布回 `build-aarch64`，用于出厂包打包：

```sh
cd /mnt/hgfs/Embedded/Gateway-zk
sh tools/build_edge_aarch64.sh --clean --publish
```

如果需要同时生成出厂初始化包：

```sh
cd /mnt/hgfs/Embedded/Gateway-zk
sh tools/build_edge_aarch64.sh --clean --publish --package --profile full
```

如果只打项目实际需要的驱动包：

```sh
cd /mnt/hgfs/Embedded/Gateway-zk
sh tools/build_edge_aarch64.sh --clean --publish --package --profile project --manifest edge-package-manifest.json
```

## KY-EMS Qt 界面编译

KY-EMS 使用目标端 Qt sysroot 加主机端 `moc/uic/rcc` 生成代码，再用 aarch64 工具链编译链接。

在 `192.168.22.11` 上执行：

```sh
cd /mnt/hgfs/Embedded/KY-EMS
sh scripts/build_ky_ems_aarch64.sh --clean
```

预期产物：

- 产物路径：`build-aarch64/KY-EMS`
- 文件类型：`ELF 64-bit LSB executable, ARM aarch64`
- 主要依赖：
  `libsqlite3`, `libQt5Charts`, `libQt5Widgets`, `libQt5Gui`, `libQt5Xml`,
  `libQt5Core`, `libGLESv2`, `libpthread`, `libstdc++`, `libc`

## KY-EMS 测试部署流程

测试边端保守替换流程如下：

```sh
mkdir -p /opt/modbus-gateway/backup/ky-ems
cp -a /opt/modbus-gateway/ky-ems/KY-EMS \
  /opt/modbus-gateway/backup/ky-ems/KY-EMS.$(date +%Y%m%d%H%M%S)
systemctl stop ky-ems.service || true
cp build-aarch64/KY-EMS /opt/modbus-gateway/ky-ems/KY-EMS
chmod 755 /opt/modbus-gateway/ky-ems/KY-EMS
systemctl start ky-ems.service
systemctl status ky-ems.service --no-pager
journalctl -u ky-ems.service -n 100 --no-pager
```

## Win 初始化包同步

KY-EMS 验证通过后，需要同步到 Win 初始化资源：

```text
D:\workspace\CloudPlatform\idea\GatewayDesktop\src\GatewayDesktop.App\Assets\DeviceInit\ky-ems\KY-EMS
```

同时替换初始化包内的同名文件：

```text
D:\workspace\CloudPlatform\idea\GatewayDesktop\src\GatewayDesktop.App\Assets\DeviceInit\gateway-factory-defaults.tar.gz
```

初始化包内的 `KY-EMS`、外层 `ky-ems/KY-EMS` 和 KY-EMS 编译产物应保持相同 SHA256。

## 注意事项

- 不要使用 Windows Qt/MinGW 的 qmake 产物编译目标端程序。
- 不要复用 RK3568 的 Qt/sysroot。
- 出厂包 `gateway-factory-defaults.tar.gz` 是边端、Win 初始化工具、平台部署的共同交付物，发布前需要校验三处 SHA256 是否一致。
