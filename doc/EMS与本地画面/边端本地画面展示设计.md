# 边端本地画面展示设计

## 目标

边端本地画面用于在网关设备接显示器时直接展示共享内存中的实时数据。它不依赖 Java 平台在线，也不依赖 MQTT Broker 在线。

当前生产方向统一为：

```text
LocalDisplay + local-kiosk.py(WebKit2) + localDisplay.screens/widgets
```

Java 平台负责配置画面组件并通过 OTA 下发 app 配置；边端 `LocalDisplay` 读取同一份配置，从共享内存取数后通过本机 HTTP/SSE 推送给 WebKit Kiosk 渲染。

## 数据链路

```text
ModbusRtu / Dlt645Driver / ComputeEngine
        |
        | 写入最新值
        v
共享内存 PointStore
        ^
        | PointStoreRouter 读取
        |
LocalDisplay
        |
        | http://127.0.0.1:18080 + SSE
        v
local-kiosk.py(WebKit2) 全屏展示

Java 平台
        |
        | 同一份 localDisplay 配置 + 当前 machineCode 的实时缓存
        v
平台“本地画面”同步预览
```

平台预览不是 VNC 截屏，而是使用同一份 `screens/widgets` 配置和平台实时缓存重新渲染。边端本地屏幕仍直接读共享内存，不依赖平台在线。

## 服务职责

- `LocalDisplay`：读取共享内存、提供本机 HTTP 页面、提供 `/api/frame` 和 `/api/stream`。
- `local-kiosk.py`：启动 WebKit2，全屏打开本机 `LocalDisplay` 页面，按配置设置 720p/全屏/缩放。
- Java 平台：配置 `localDisplay.screens/widgets`，提供同屏预览，随配置 OTA 一起下发。

不做：

- 不负责采集。
- 不替代 `MqttDriver`。
- 不执行告警计算。
- 不直接执行远程命令。

## 配置结构

核心配置在 app 配置的 `localDisplay` 下：

```json
{
  "localDisplay": {
    "enabled": true,
    "renderer": "webkit",
    "bindHost": "127.0.0.1",
    "port": 18080,
    "refreshIntervalMs": 1000,
    "maxPointsPerFrame": 500,
    "showOnlyConfiguredPoints": true,
    "sharedMemoryNames": ["gateway_point_store"],
    "kiosk": {
      "width": 1280,
      "height": 720,
      "fullscreen": true,
      "setDisplayMode": true,
      "enabled": true,
      "requireDisplayConnected": true,
      "requireEdid": true,
      "displayOutput": "",
      "zoom": 1.0
    },
    "screens": [
      {
        "screenCode": "overview",
        "title": "设备总览",
        "layout": {
          "type": "grid",
          "columns": 12,
          "rowHeight": 64,
          "gap": 12
        },
        "widgets": [
          {
            "id": "online_status",
            "type": "statusCard",
            "title": "设备在线",
            "bind": {"index": 11000},
            "valueFormat": "boolOnline",
            "position": {"x": 0, "y": 1, "w": 3, "h": 2}
          },
          {
            "id": "key_points",
            "type": "pointTable",
            "title": "关键点表",
            "bind": {"indexes": [11000, 11001, 11002]},
            "columns": ["index", "meterCode", "pointCode", "name", "value", "unit", "quality", "time"],
            "position": {"x": 0, "y": 3, "w": 12, "h": 6}
          }
        ]
      }
    ],
    "viewTemplate": {
      "enabled": false,
      "refreshIntervalMs": 5000
    }
  }
}
```

## 组件类型

- `groupTitle`：标题/分区文本。
- `statusCard`：单点状态卡片，适合设备在线、通讯状态、DI 状态。
- `valueCard`：单点数值卡片，适合电压、电流、功率等关键值。
- `pointTable`：多点表格，适合批量点位。
- `alarmSummary`：告警摘要表格，展示格式和表格一致，后续可接入告警筛选。

组件通用字段：

- `id`：组件唯一标识。
- `type`：组件类型。
- `title`：组件标题。
- `bind.index`：单点组件绑定的点位 index。
- `bind.indexes`：多点组件绑定的点位 index 列表。
- `position.x/y/w/h`：12 栅格布局中的列、行、宽、高。
- `columns`：表格组件显示列。
- `valueFormat`：单点值格式，支持 `number`、`raw`、`boolOnline`。

## Java 配置方式

Java 平台“本地画面”页面提供组件化配置器：

- 新增/删除画面 `screen`。
- 配置画面标题、列数、行高、间距。
- 添加状态卡片、数值卡片、表格、标题等组件。
- 通过下拉选择点位，避免手输 index 错误。
- 平台同屏预览直接按当前组件配置渲染。
- 保存 app 配置后可通过“生成并发布 OTA”下发到边端。

## 边端运行方式

出厂默认配置中 `localDisplay.enabled=false`，不主动启动本地画面，避免没有屏幕的设备常驻 WebKit 占用内存。

当 `monitor-service.json` 中 `localDisplay.enabled=true` 时，服务管理脚本启动本机 HTTP 服务：

```text
local-display@monitor-service.service
```

当 `localDisplay.kiosk.enabled` 未显式关闭，且检测到显示输出后，再启动全屏 WebKit：

```text
local-kiosk@monitor-service.service
```

显示检测规则：

- 默认 `requireDisplayConnected=true`，无显示输出时跳过 `local-kiosk`。
- 默认 `requireEdid=true`，需要 DRM connector 的 EDID 非空才认为屏幕真实插入。
- 部分板卡 HDMI 通过 `DPI-1` 暴露且没有 EDID，此时应显式配置 `"displayOutput": "DPI-1"` 和 `"requireEdid": false`。
- `local-kiosk@.service` 配置了 `MemoryHigh=384M`、`MemoryMax=512M`，WebKit 异常膨胀时由 systemd 兜底重启。

本机访问地址：

```text
http://127.0.0.1:18080/
```

调试接口：

- `/api/health`：服务状态。
- `/api/pages`：画面列表。
- `/api/points`：可路由点位元数据。
- `/api/frame?pageCode=overview`：单帧数据。
- `/api/stream?pageCode=overview`：SSE 实时推送。

## OTA 生效流程

1. Java 平台编辑 `localDisplay.screens/widgets`。
2. 保存 app 配置。
3. 生成配置 OTA 包。
4. 发布到 `edge/ota/request/{machineCode}`。
5. 边端下载并应用配置包。
6. `gateway-services.sh` 根据 app 配置重启 `local-display` 和 `local-kiosk`。
7. WebKit 页面重新加载新配置。

## 生产约束

- `bindHost` 默认保持 `127.0.0.1`，避免本地展示端口暴露到外网。
- 单屏组件绑定点位不宜超过 `maxPointsPerFrame`。
- 大量点位优先用分页/多屏拆分，不建议一个表格塞满所有点位。
- 本地画面只读展示。若后续需要按钮写设备，应通过共享内存写队列和驱动写回机制实现，并增加权限控制。
