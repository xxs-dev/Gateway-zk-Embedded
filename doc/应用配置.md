
# App Config

`device.json` 负责设备、从站、点位、共享内存、SQLite。

`app config` 负责网关级运行参数，例如 MQTT、实时订阅和 OTA。

字段说明：

- `identityConfigFile`: 网关身份配置文件路径，默认 `/opt/modbus-gateway/config/runtime/device_identity.json`。
- `deviceConfigFiles`: 需要加载的一个或多个 `device.json` 路径。
- `mqtt.telemetryTopic`: 测点数据上传基础 topic，实际发布到 `<telemetryTopic>/<machineCode>`。
- `mqtt.changeEventTopic`: 变位事件上传基础 topic，实际发布到 `<changeEventTopic>/<machineCode>`。
- `mqtt.alarmTopic`: 告警事件上传基础 topic，实际发布到 `<alarmTopic>/<machineCode>`。
- `mqtt.statusTopic`: 状态类消息基础 topic，实际发布到 `<statusTopic>/<machineCode>`，供实时监控或状态同步使用。
- `mqtt.commandRequestTopic`: 下行写入命令基础 topic。运行时会自动订阅 `<commandRequestTopic>/<machineCode>`，避免其他网关接收无关命令。
- `mqtt.commandReplyTopic`: 写入命令回执基础 topic，实际发布到 `<commandReplyTopic>/<machineCode>`。
- `mqtt.otaRequestTopic`: OTA 请求基础 topic。平台发布时会自动发送到 `<otaRequestTopic>/<machineCode>`，边端也只订阅自己的机器级 topic。
- `mqtt.tls.*`: 内置 MQTT TLS 配置。`tls.enabled=true` 或 broker 使用 `ssl://` / `tls://` / `mqtts://` 时启用 TLS；需要边端存在 OpenSSL 运行库。`certFile` 和 `keyFile` 支持双向 TLS，必须成对配置。
- `mqttDriver.sharedMemoryNames`: 默认只包含 `gateway_point_store`，所有协议驱动统一写入该共享内存。
- `ota.downloadRetryCount` / `ota.downloadRetryBackoffMs`: OTA 下载失败后的重试次数和退避时间。
- `mqtt.otaReplyTopic`: OTA 受理结果基础 topic，实际发布到 `<otaReplyTopic>/<machineCode>`。
- `mqtt.otaStatusTopic`: OTA 过程状态基础 topic，实际发布到 `<otaStatusTopic>/<machineCode>`。
- `mqtt.systemMonitorRequestTopic`: 系统监测订阅基础 topic。运行时自动订阅 `<systemMonitorRequestTopic>/<machineCode>`。
- `mqtt.systemMonitorReplyTopic`: 系统监测回执基础 topic，实际发布到 `<systemMonitorReplyTopic>/<machineCode>`。
- `mqtt.systemMonitorTelemetryTopic`: 系统资源上报基础 topic，实际发布到 `<systemMonitorTelemetryTopic>/<machineCode>`。
- `mqtt.systemMonitorAlertTopic`: 系统资源告警基础 topic，实际发布到 `<systemMonitorAlertTopic>/<machineCode>`。
- `mqtt.systemMonitorPointTopic`: 共享内存点查询回传基础 topic，实际发布到 `<systemMonitorPointTopic>/<machineCode>`。
- `mqtt.diagRequestTopic`: 远程诊断请求基础 topic。运行时自动订阅 `<diagRequestTopic>/<machineCode>`。
- `mqtt.diagReplyTopic`: 远程诊断回执基础 topic，实际发布到 `<diagReplyTopic>/<machineCode>`。
- `mqtt.configPullRequestTopic`: 配置拉取请求基础 topic。运行时自动订阅 `<configPullRequestTopic>/<machineCode>`。
- `mqtt.configPullReplyTopic`: 配置拉取回传基础 topic，实际发布到 `<configPullReplyTopic>/<machineCode>`。
- `mqttDriver`: 共享内存扫描和 MQTT 上传驱动配置。
- `alarmStore`: 告警落库配置。
- `realtime`: 实时监测侧消费 topic 和内存缓存约束配置。
- `localDisplay`: 边端本地屏幕展示配置。出厂默认关闭；在 `monitor-service.json` 中启用后启动 `local-display@monitor-service.service`，并在 `kiosk.enabled=true` 且显示输出检测通过时启动本机全屏 `local-kiosk@monitor-service.service`。默认访问地址为 `http://127.0.0.1:18080/`；Java 平台也用同一配置进行同步预览。
- `cameraService`: 摄像头推流配置。出厂默认关闭；只有 `cameraService.enabled=true` 且至少一个 `cameras[].enabled=true` 时才启动 `camera-service@camera-service.service`。关闭时不注册摄像头共享内存状态点，Java 平台摄像头监控接口返回空 `streams` 且不查询 ZLMediaKit。视频流由 `ffmpeg` 推送到平台媒体服务，在线、fps、码率、错误码状态写入共享内存并发布到 `<cameraService.statusTopic>/<machineCode>`。
- `cameraService.media.serverUrl`: 平台媒体服务入口。公网部署时填写平台固定公网 IP 或域名，例如 `rtsp://203.0.113.10:8554`；边端主动出站推流，平台不主动访问边端内网地址。
- `cameraService.media.auth`: 平台媒体入口认证。`basic` 会自动注入 RTSP userinfo，`token_query` 会追加 token 参数；状态上报默认隐藏凭据。
- `cameraService.cameras[].sourceAuth`: 现场摄像头源认证。摄像头和边端在同一局域网内，边端拉取摄像头 RTSP 时使用该认证；不要和平台推流入口认证混用。
- `ota`: OTA 升级执行配置，包括本地目录、脚本、超时和对象存储参数。

说明：

- 所有 topic 字段在 JSON 里都只保存基础 topic，不需要手工拼 `machineCode`。
- `machineCode` 由 `identityConfigFile` 指向的 `device_identity.json` 提供；设备 JSON 顶层不再维护 `machineCode`。
- Java 平台发布下行请求时，必须发布到 `<baseTopic>/<machineCode>`。
- 边端 `MqttDriver / SystemMonitor` 也只会订阅自己 `machineCode` 对应的请求 topic，并把所有上行消息发布到 `<baseTopic>/<machineCode>`。
- 这样 OTA、写命令、系统监测订阅、诊断命令都不会再广播给同 broker 上的全部网关。
- `ota.storage.provider`: 当前支持 `local` / `minio` 配置解析，并在 OTA 下载阶段参与制品地址解析。
- `ota.storage.minio.publicBaseUrl`: 对外直链基地址，留空则由上层按 endpoint/bucket 拼装。

当前 C++ 侧能力：

- 已支持读取独立 `app config`
- 已支持独立 `telemetry/alarm/status/command/ota` topic 配置
- 已支持 `realtime` 与 `ota` 配置段解析
- 已支持 MQTT 告警发布走 `alarmTopic`
- 已支持 OTA 下载、校验、安装、失败回滚和状态上报
- 已支持 `LocalDisplay` 从共享内存读取点位并提供本地 HTTP/SSE 只读画面
- 已支持 Python GTK `local-kiosk` 在边端桌面全屏展示
- 已支持 `CameraService` 托管 USB/RTSP/test 视频源并通过 `ffmpeg` 推 RTSP 到平台媒体服务
- Java 平台已支持按同一 `localDisplay` 配置和当前 `machineCode` 的实时 MQTT 缓存同步预览
