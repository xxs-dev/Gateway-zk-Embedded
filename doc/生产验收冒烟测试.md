# 生产验收冒烟测试

本文用于边端网关正式部署、OTA 升级后、现场故障恢复后的快速验收。目标是用一个脚本检查“能否安全进入生产运行”，不是替代长稳压测。

脚本路径：

```sh
/opt/modbus-gateway/bin/production-smoke-test.sh
```

推荐入口：

```sh
/opt/modbus-gateway/bin/gateway-run.sh smoke
```

## 1. 适用场景

- 新设备出厂配置完成后。
- 现场更换 MQTT broker、TLS 证书、串口配置后。
- 平台下发配置 OTA 后。
- 边端异常断电、重启、共享内存清理后。
- 正式生产前最后一轮检查。

## 2. 执行方式

默认检查：

```sh
/opt/modbus-gateway/bin/gateway-run.sh smoke
```

指定安装目录：

```sh
GATEWAY_HOME=/opt/modbus-gateway /opt/modbus-gateway/bin/production-smoke-test.sh
```

启用主动 MQTT 连接测试：

```sh
MQTT_CONNECT_TEST=1 /opt/modbus-gateway/bin/gateway-run.sh smoke
```

主动 MQTT 测试会执行 `MqttDriver --once`，可能向 broker 发布一次按需数据。生产现场默认不启用，避免误触发第三方平台数据处理。

## 3. 检查项

| 检查项 | 生产意义 | 失败处理 |
| --- | --- | --- |
| 身份配置 | 确认 `machineCode` 是平台登记的唯一网关标识，避免 topic 串台 | 修改 `config/runtime/device_identity.json` 后重启服务 |
| 运行文件 | 确认可执行文件和脚本齐全，避免 OTA 包漏文件 | 重新安装 OTA 包或执行出厂安装脚本 |
| 设备配置引用 | 确认 `deviceConfigFiles[]` 指向的 JSON 都存在 | 在平台重新生成配置包，或修正 app 配置中的路径 |
| MQTT TLS | 确认 TLS 已启用、OpenSSL 运行库存在、CA/客户端证书路径正确；本机 stunnel 方案还会校验 `accept/connect/checkHost/CAfile` | 补齐 `libssl/libcrypto`、CA 文件，修正 broker scheme 或替换 stunnel 模板占位 |
| systemd 服务 | 确认需要运行的协议驱动、事件引擎、MQTT、监测服务处于 active | 查看 `journalctl`，修复配置后 `systemctl restart gateway-services.service` |
| 共享内存 | 确认 `pointctl stats/pending` 可返回，不存在锁死或写队列异常 | 停服务后清理 `/dev/shm/gateway_point_store*` 再启动 |
| 磁盘空间 | 防止 SQLite、MQTT ring、OTA 包把磁盘打满 | 清理旧 OTA 包、日志、无效缓存，或扩容 |
| 配置权限 | 避免 MQTT 密码、TLS 路径、身份信息被普通用户读取 | 建议 `chmod 600/640` |
| OTA 脚本 | 确认升级、回滚脚本存在且可执行 | 重新部署 `ota-apply.sh` 和 `ota-rollback.sh` |
| MQTT 主动连接 | 可选确认 broker 实际可连接 | 检查网络、账号密码、TLS 证书和 broker 端 ACL |

## 4. 通过标准

脚本最后输出：

```text
summary: pass=... warn=... fail=0
```

验收标准：

- `fail=0` 才允许进入生产。
- `warn` 允许存在，但必须人工确认原因。
- 生产跨公网或不可信网络时，非 TLS broker 警告不能忽略。
- `tls.insecureSkipVerify=true` 会被判定为失败。
- `ota.retentionCount > 3` 会被判定为失败。

## 5. 常见输出说明

`[PASS] pointctl stats completed`

说明共享内存可以被正常打开，内部互斥锁没有卡死。

`[FAIL] service not active: modbus-rtu@ttySP1.service`

说明当前配置要求这个串口驱动运行，但 systemd 中不是 active。优先看：

```sh
journalctl -u modbus-rtu@ttySP1.service -n 100 --no-pager
```

`[WARN] mqtt-service.json uses non-TLS broker`

说明 app 配置仍使用 `tcp://` broker。仅允许内网临时测试；跨公网或不可信网络必须改为 `ssl://`、`tls://`、`mqtts://` 或 `tls.enabled=true`。

`[FAIL] TLS CA file missing`

说明启用了 TLS 但 CA 文件路径不存在。把 CA 放到配置声明路径，或修正 `tls.caFile`。

`[FAIL] stunnel connect still uses template placeholder`

说明 app broker 已指向本机 stunnel 端口，但 `config/runtime/tls/*-stunnel.conf` 仍是出厂模板。必须把 `connect`、`checkHost` 和 `CAfile` 改成真实生产 broker 与证书路径。

`[FAIL] pointctl stats failed or timed out`

说明共享内存可能损坏、旧进程未退出或互斥锁异常。处理方式：

```sh
systemctl stop gateway-services.service
/opt/modbus-gateway/bin/gateway-services.sh stop
rm -f /dev/shm/gateway_point_store*
systemctl start gateway-services.service
```

不要在服务运行时删除共享内存。

## 6. 可调参数

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `GATEWAY_HOME` | `/opt/modbus-gateway` | 边端安装目录 |
| `APP_CONFIG` | `$GATEWAY_HOME/config/runtime/apps/mqtt-service.json` | MQTT 服务 app 配置 |
| `MONITOR_CONFIG` | `$GATEWAY_HOME/config/runtime/apps/monitor-service.json` | 系统监测 app 配置 |
| `IDENTITY_CONFIG` | `$GATEWAY_HOME/config/runtime/device_identity.json` | 网关身份配置 |
| `MIN_FREE_MB` | `512` | 最小剩余磁盘空间 |
| `MAX_DISK_USED_PERCENT` | `85` | 最大磁盘使用率 |
| `MQTT_CONNECT_TEST` | `0` | 是否执行主动 MQTT 连接测试 |

## 7. 与长稳压测的边界

冒烟测试只判断“基础运行条件是否满足”。以下内容仍需要单独压测或现场验证：

- 8 路 485 同时采集时的 CPU、内存、串口时序。
- MQTT 断链 24 小时后的 ring/outbox 补传能力。
- OTA 下载中断、断电、脚本失败、回滚路径。
- TLS 双向证书在真实生产 broker 上的验收。
- 72 小时长稳运行后的磁盘、SQLite、日志增长。
