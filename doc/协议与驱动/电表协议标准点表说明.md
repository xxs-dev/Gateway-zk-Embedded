# DLT645-2007 标准点表说明

本文说明当前 `DLT645-2007` 标准点表模板的用途、字段和维护规则。标准模板文件位于：

- [dlt645_2007_standard_points.json](../../config/templates/dlt645_2007_standard_points.json)

## 当前定位

标准点表是 DLT645 电表配置的固定基线：

- 模板由项目维护，不要求现场人员逐点手填 DI。
- 平台和 Windows 工具生成 DLT645 设备配置时使用同一份模板。
- 边端 `Dlt645Driver` 在 `meters[].points` 为空时可按模板自动展开。
- 模板用于统一中文名称、`pointCode`、`DI`、倍率、单位和数据类型。

模板不是运行时采集结果，也不保存具体电表地址和最终 `index`。具体电表实例仍在设备配置的 `meters[]` 中维护。

## 文件结构

模板文件结构：

```json
{
  "protocol": "dlt645_2007",
  "version": "1.1.0",
  "source": "national_standard",
  "points": [
    {
      "templateKey": "forward_active_energy_total",
      "pointCode": "forward_active_energy_total",
      "name": "正向有功总电能",
      "category": "telemetry",
      "di": "00010000",
      "dataType": "dlt645_bcd",
      "byteCount": 4,
      "scale": 0.01,
      "unit": "kWh",
      "defaultIntervalMs": 1000,
      "storeLatest": true,
      "storeHistory": true,
      "reportOnChange": false,
      "enabledByDefault": true
    }
  ]
}
```

## 字段说明

| 字段 | 说明 |
| --- | --- |
| `templateKey` | 模板内唯一键，用于生成和差异比对 |
| `pointCode` | 业务点编码，导出到设备配置后保持稳定 |
| `name` | 中文点名，实时监控和表格编辑默认显示 |
| `category` | 点位分类，常见 `telemetry`、`status`、`event` |
| `di` | DLT645 数据标识，4 字节十六进制字符串 |
| `dataType` | 解析类型 |
| `byteCount` | 响应数据字节数 |
| `scale` | 倍率 |
| `unit` | 单位 |
| `defaultIntervalMs` | 默认采集周期 |
| `storeLatest` | 是否写最新值缓存 |
| `storeHistory` | 是否进入历史采样 |
| `reportOnChange` | 是否按变位事件上传 |
| `enabledByDefault` | 生成设备配置时是否默认启用 |

可选字段：

- `desc`
- `phase`
- `tariffNo`
- `precision`
- `alarmSuggested`
- `notes`

## 数据类型

当前模板使用的数据类型包括：

- `dlt645_bcd`：常规 BCD 数值
- `dlt645_bcd_signed`：有符号 BCD 数值
- `dlt645_uint32`：无符号整数
- `dlt645_ascii`：文本
- `dlt645_datetime`：日期时间
- `dlt645_status_bits`：状态位
- `device_online`：驱动生成的在线状态点

## 点位范围

当前模板覆盖以下常用国标点位类别：

- 正向/反向/组合有功电能
- 正向/反向/四象限无功电能
- 尖峰平谷分时电能
- 上 1 到上 12 个结算日电能
- 三相电压
- 三相电流
- 总及分相有功功率
- 总及分相无功功率
- 总及分相视在功率
- 总及分相功率因数
- 电网频率
- 最大需量及发生时间
- 电表状态类点
- 驱动补充在线点 `device_online`

## 结算日电能规则

结算日数据按规则展开，不手工维护 240 条重复逻辑。

组合规则：

```text
dataId = typeCode | (itemNo << 8) | dayNo
```

参数：

| 参数 | 取值 |
| --- | --- |
| `dayNo` | `1..12`，表示上 1 到上 12 个结算日 |
| `itemNo` | `0` 总电量，`1..4` 费率 1 到费率 4 |
| 正向有功 `typeCode` | `0x00010000` |
| 反向有功 `typeCode` | `0x00020000` |
| 正向无功 `typeCode` | `0x00030000` |
| 反向无功 `typeCode` | `0x00040000` |

示例：

- 上 1 个结算日正向有功总电量：`00010001`
- 上 1 个结算日正向有功费率 1 电量：`00010101`
- 上 12 个结算日反向无功费率 4 电量：`0004040C`

## 命名规则

`pointCode` 使用英文稳定键，中文名用于界面显示。

常用命名：

| 类别 | 示例 |
| --- | --- |
| 电能 | `forward_active_energy_total` |
| 分时电能 | `forward_active_energy_tariff_1` |
| 结算日 | `settlement_day_1_forward_active_energy_total` |
| 电压 | `voltage_a`、`voltage_b`、`voltage_c` |
| 电流 | `current_a`、`current_b`、`current_c` |
| 有功功率 | `active_power_total`、`active_power_a` |
| 无功功率 | `reactive_power_total`、`reactive_power_a` |
| 视在功率 | `apparent_power_total`、`apparent_power_a` |
| 功率因数 | `power_factor_total`、`power_factor_a` |
| 频率 | `grid_frequency` |
| 在线点 | `device_online` |

结算日命名格式：

```text
settlement_day_{day}_{kind}_{item}
```

其中：

- `{day}`：`1..12`
- `{kind}`：`forward_active_energy`、`reverse_active_energy`、`forward_reactive_energy`、`reverse_reactive_energy`
- `{item}`：`total`、`tariff_1`、`tariff_2`、`tariff_3`、`tariff_4`

## 默认行为

| 点位类别 | `storeLatest` | `storeHistory` | `reportOnChange` |
| --- | --- | --- | --- |
| 电能类 | true | true | false |
| 结算日电能类 | true | true | false |
| 电压/电流/功率/频率类 | true | true | false |
| 状态类 | true | false | true |
| 在线点 | true | false | true |

## index 分配

标准模板本身不固定 `index`。设备配置生成时再按电表实例分配：

- 第 1 块表：`200000 + 点位偏移`
- 第 2 块表：`210000 + 点位偏移`
- 第 3 块表：`220000 + 点位偏移`
- 每块表预留 `10000` 个 index

正式工程生成时，应再经过全局点位路由表检查，确保不会和 Modbus、DIDO、CAN、IEC、EMS 虚拟点等重复。
