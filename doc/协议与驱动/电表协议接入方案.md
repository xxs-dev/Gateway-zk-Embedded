# DLT645-2007 电表协议接入说明

本文说明当前边端 `DLT645-2007` 电表采集能力的落地状态、运行配置和边界。早期规划内容已收敛为当前实现说明，后续新项目按本文和标准点表模板执行。

## 当前状态

当前已落地能力：

- 独立可执行程序：`Dlt645Driver`
- systemd 模板：`dlt645-driver@.service`
- 协议类型：`protocol.type=dlt645_2007`
- 串口链路：支持常见 `2400 8E1` 参数，可按现场电表调整
- 多电表地址：同一配置文件可配置多个 `meters[]`
- 标准点表模板：`config/templates/dlt645_2007_standard_points.json`
- 空 `meters[].points` 自动展开：运行时按标准模板生成点位
- 共享内存接入：采集结果写入 `gateway_point_store`
- MQTT 接入：由 `MqttDriver` 统一读取共享内存并上送
- 实时监测接入：平台和 Windows 工具按统一点位模型展示
- 设备在线点：驱动补充 `device_online`，用于在线/离线判断
- 帧间隔策略：同一电表地址按 `frameIntervalMs` 控制，不同电表地址不互相等待

当前边界：

- DLT645 按只读电表采集协议使用，不提供写入能力。
- DLT645 不依托 Modbus 驱动实现，和 Modbus 一样都是独立协议驱动。
- 视频、EMS 策略、DIDO 等能力不属于 DLT645 驱动职责。

## 运行链路

```text
DLT645 电表
  -> Dlt645Driver
  -> MemoryPointStore / gateway_point_store
  -> MqttDriver
  -> 平台实时监测 / 全量上传 / 告警 / Windows 工具
```

协议差异只保留在采集和解析层。对上统一输出点位值：

- `index`
- `machineCode`
- `meterCode`
- `pointCode`
- `name`
- `value`
- `quality`
- `ts`
- `expireAt`

因此 MQTT、平台实时监控、Windows 实时监控和本地画面不需要区分 Modbus 或 DLT645。

## 配置文件

当前样例文件：

- [device_dlt645_multi_meter_1_2.json](../../config/runtime/devices/device_dlt645_multi_meter_1_2.json)
- [dlt645_2007_standard_points.json](../../config/templates/dlt645_2007_standard_points.json)
- [mqtt-service.json](../../config/runtime/apps/mqtt-service.json)

最小设备配置结构：

```json
{
  "schemaVersion": "1.0.0",
  "protocol": {
    "type": "dlt645_2007",
    "transport": {
      "serialPort": "/dev/ttySP1",
      "baudRate": 2400,
      "dataBits": 8,
      "stopBits": 1,
      "parity": "E",
      "timeoutMs": 1000,
      "frameIntervalMs": 200
    },
    "standardPointsFile": "config/templates/dlt645_2007_standard_points.json",
    "standardPointsVersion": "1.1.0"
  },
  "collect": {
    "defaultIntervalMs": 1000,
    "batchOptimize": false,
    "maxBatchRegisters": 1
  },
  "memoryStore": {
    "enabled": true,
    "sharedMemoryName": "gateway_point_store"
  },
  "meters": [
    {
      "meterCode": "METER0001",
      "deviceName": "1号电表",
      "address": "000000000001",
      "points": []
    }
  ]
}
```

说明：

- `meters[].address` 是 12 位 DLT645 表地址。
- `meters[].points` 为空时，驱动按标准点表自动展开。
- 如果现场只需要部分点，也可以在 `points[]` 中写实例化后的点位清单。
- `machineCode` 统一来自 `config/runtime/device_identity.json`，设备配置文件不再维护网关身份。

## 点位模型

DLT645 点位不使用 Modbus 的 `function/address/length` 语义，读配置使用 `read.dlt645`：

```json
{
  "index": 200001,
  "pointCode": "forward_active_energy_total",
  "name": "正向有功总电能",
  "category": "telemetry",
  "enabled": true,
  "isStore": true,
  "reportOnChange": false,
  "persistIntervalSec": 60,
  "read": {
    "enable": true,
    "dataType": "dlt645_bcd",
    "intervalMs": 1000,
    "unit": "kWh",
    "dlt645": {
      "di": "00010000",
      "byteCount": 4,
      "scale": 0.01
    }
  },
  "write": {
    "enable": false
  }
}
```

当前支持的数据类型以标准模板为准，常用类型包括：

- `dlt645_bcd`
- `dlt645_bcd_signed`
- `dlt645_uint32`
- `dlt645_ascii`
- `dlt645_datetime`
- `dlt645_status_bits`
- `device_online`

## 标准点表策略

DLT645 的点表天然由国标数据标识 `DI` 决定。当前采用固定标准模板，不要求用户逐个手工配置 DI：

- 标准模板版本化维护在 `config/templates/dlt645_2007_standard_points.json`。
- Windows 工具和平台配置页生成 DLT645 设备时，应基于该模板展开点位。
- 边端运行时如果发现 `meters[].points` 为空，也会基于模板自动展开。
- 标准模板只定义点位语义，不固定具体 `index`；实例化时再分配全局唯一 index。

标准点表的详细字段和命名规则见 [电表协议标准点表说明.md](电表协议标准点表说明.md)。

## 自动展开与 index 分配

当前 DLT645 多表自动展开规则：

- 第 1 块表：`200000 + 点位偏移`
- 第 2 块表：`210000 + 点位偏移`
- 第 3 块表：`220000 + 点位偏移`
- 之后每块表预留 `10000` 个 index

Windows 工具和平台端生成正式工程时，仍应执行全局点位检查：

- `index` 全局唯一
- `meterCode + pointCode` 在同一工程内可定位唯一业务点
- EMS 模式不自动重排策略已引用的 index
- 网关模式可在全局检查中使用一键排序修复重复 index

## 调度策略

DLT645 不适合像 Modbus 一样按连续寄存器合并块读。当前调度规则：

- 一次请求一个 `DI`
- 同一电表地址按点位到期时间轮询
- 同一电表地址遵守 `frameIntervalMs`
- 不同电表地址不互相套用单地址帧间隔
- 点位采集周期优先使用 `read.intervalMs`，否则使用 `collect.defaultIntervalMs`

该策略优先保证稳定性和可排障性。后续如确有性能压力，再按具体表型评估冻结类或连续类 DI 的批量优化。

## 写入边界

当前 DLT645 驱动按只读电表协议验收：

- `pointctl write` 不用于 DLT645 点位。
- MQTT 控制命令不下发到 DLT645。
- 标准模板内点位默认 `write.enable=false`。
- 如果后续确需电表参数设置，应作为单独需求评审密码、操作者代码、权限和现场风险。

## 与平台和 Windows 工具的关系

平台侧重点：

- 保存 DLT645 设备配置。
- 使用标准模板生成或预览点表。
- 做全局点位检查。
- 通过配置 OTA 下发到边端。
- 通过 MQTT 全量/实时链路监测数据。

Windows 工具侧重点：

- 本地新建或拉取工程。
- 表格化编辑电表基础信息和点表。
- 直连或 MQTT 写回配置。
- 生成初始化包和配置 OTA 包。
- 做现场联调和实时监测。

两端都应复用同一份标准点表模板，避免平台和 Windows 工具生成不同口径的 DI、倍率、单位和点名。

## 运行与验证

手工运行：

```bash
/opt/modbus-gateway/bin/Dlt645Driver \
  --config /opt/modbus-gateway/config/runtime/devices/device_dlt645_multi_meter_1_2.json \
  --app-config /opt/modbus-gateway/config/runtime/apps/mqtt-service.json
```

通过 systemd 运行：

```bash
systemctl status dlt645-driver@device_dlt645_multi_meter_1_2.service
journalctl -u dlt645-driver@device_dlt645_multi_meter_1_2.service -n 100 --no-pager
```

共享内存与 MQTT 验证见 [电表协议运行测试.md](电表协议运行测试.md)。
