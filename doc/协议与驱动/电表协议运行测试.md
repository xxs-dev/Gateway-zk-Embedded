# DLT645-2007 运行测试说明

本文用于验证 DLT645 电表从采集到共享内存、MQTT 和平台实时监测的完整链路。

## 测试范围

当前验证对象：

- `Dlt645Driver` 配置解析
- 标准点表自动展开
- 多电表地址轮询
- 串口帧间隔控制
- 共享内存写入
- MQTT 全量/实时上传
- 平台或 Windows 工具实时监控显示

当前不验证：

- DLT645 写入
- 特殊表型私有 DI
- 电表参数设置

## 测试文件

- 设备配置：[device_dlt645_multi_meter_1_2.json](../../config/runtime/devices/device_dlt645_multi_meter_1_2.json)
- MQTT 配置：[mqtt-service.json](../../config/runtime/apps/mqtt-service.json)
- 标准点表：[dlt645_2007_standard_points.json](../../config/templates/dlt645_2007_standard_points.json)

## 启动方式

手工 mock 验证：

```bash
cd /opt/modbus-gateway
./bin/Dlt645Driver \
  --config config/runtime/devices/device_dlt645_multi_meter_1_2.json \
  --app-config config/runtime/apps/mqtt-service.json \
  --mock
./bin/MqttDriver --app-config config/runtime/apps/mqtt-service.json
```

真实电表验证：

```bash
cd /opt/modbus-gateway
./bin/Dlt645Driver \
  --config config/runtime/devices/device_dlt645_multi_meter_1_2.json \
  --app-config config/runtime/apps/mqtt-service.json
./bin/MqttDriver --app-config config/runtime/apps/mqtt-service.json
```

systemd 验证：

```bash
systemctl status dlt645-driver@device_dlt645_multi_meter_1_2.service
journalctl -u dlt645-driver@device_dlt645_multi_meter_1_2.service -n 100 --no-pager
systemctl status mqtt-driver@mqtt-service.service
```

## 预期行为

启动后应看到：

1. `Dlt645Driver` 识别 `protocol.type=dlt645_2007`。
2. `meters[]` 中每块电表地址被加载。
3. 空 `points[]` 按标准点表自动展开。
4. 每个点按 `read.dlt645.di` 逐点读取。
5. 成功值写入 `gateway_point_store`。
6. `MqttDriver` 从共享内存读取并发布。
7. 实时监控可按 `meterCode`、`pointCode` 和中文点名展示。

## index 预期

自动展开时，每块表预留 `10000` 个 index：

| 电表序号 | index 段 |
| --- | --- |
| 第 1 块表 | `200000` 起 |
| 第 2 块表 | `210000` 起 |
| 第 3 块表 | `220000` 起 |

不同电表即使 `pointCode` 相同，也应通过 `meterCode + index` 区分，不应互相覆盖。

## MQTT 订阅

实时数据：

```bash
mosquitto_sub -h <broker> -p <port> -t 'edge/telemetry/+' -v
```

全量数据：

```bash
mosquitto_sub -h <broker> -p <port> -t 'edge/telemetry/full/+' -v
```

状态：

```bash
mosquitto_sub -h <broker> -p <port> -t 'edge/status/+' -v
```

关键字段：

- `machineCode`
- `meters[].meterCode`
- `meters[].values[].index`
- `meters[].values[].pointCode`
- `meters[].values[].name`
- `meters[].values[].value`
- `meters[].values[].quality`

## 共享内存检查

使用 `pointctl` 查看点位：

```bash
/opt/modbus-gateway/bin/pointctl --store gateway_point_store list --limit 20
```

检查重点：

- 两个 `meterCode` 都有数据。
- `quality=1` 表示最近一次采集成功。
- `device_online=1` 表示该电表最近采集成功。
- `ts` 持续刷新。
- `index` 不重复。

## 串口和电表地址排障

如果没有数据，优先检查：

```bash
ls -l /dev/ttySP*
stty -F /dev/ttySP1 -a
journalctl -u dlt645-driver@device_dlt645_multi_meter_1_2.service -n 200 --no-pager
```

常见原因：

- `meters[].address` 不是现场 12 位表地址。
- 串口号错误。
- 波特率、校验位、停止位错误。
- 485 A/B 接反。
- 同一串口被其他服务占用。
- 电表不支持模板中某些冷门 DI。

## 标准点表排障

如果进程启动失败或点表为空，检查：

- `protocol.standardPointsFile`
- `dlt645.standardPointsFile`
- `config/templates/dlt645_2007_standard_points.json` 是否随配置包下发
- JSON 是否可解析
- `points[]` 是否被错误生成成空对象或空行

## 验收标准

一次 DLT645 联调至少满足：

- 驱动无崩溃、无持续重启。
- 至少一个真实电表 `device_online=1`。
- 至少一组电能、电压或电流点 `quality=1`。
- 共享内存能查到对应 `meterCode`。
- MQTT 实时或全量 topic 能收到对应 `meterCode`。
- 平台或 Windows 工具能显示中文点名和值。

## 当前限制

- DLT645 当前不支持写入。
- 状态字当前以基础点位呈现，细粒度位含义需按表型补充。
- 冻结类和特殊表型 DI 需要结合现场设备说明书复核。
- mock 模式只验证流程，不代表真实电表值。
