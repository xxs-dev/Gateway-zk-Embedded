
# Ubuntu 20.04 ARM 部署说明

## 1. 目录建议

推荐部署目录：

- 基础程序：`/opt/modbus-gateway/bin/SystemMonitor`
- 基础程序：`/opt/modbus-gateway/bin/MqttDriver`
- 基础程序：`/opt/modbus-gateway/bin/pointctl`
- 项目驱动：`/opt/modbus-gateway/bin/ModbusRtu`
- 项目驱动：`/opt/modbus-gateway/bin/Dlt645Driver`
- 项目驱动：`/opt/modbus-gateway/bin/DioDriver`
- 项目驱动：`/opt/modbus-gateway/bin/CanDriver`
- 项目驱动：`/opt/modbus-gateway/bin/IecDriver`
- 可选服务：`/opt/modbus-gateway/bin/EventEngine`
- 可选服务：`/opt/modbus-gateway/bin/ComputeEngine`
- 设备配置：`/opt/modbus-gateway/config/runtime/devices/*.json`
- 应用配置：`/opt/modbus-gateway/config/runtime/apps/*.json`
- 交付清单：`/opt/modbus-gateway/config/runtime/edge-package-manifest.json`
- 本地数据：`/opt/modbus-gateway/data/`

当前已存在的所有驱动和边端组件统一按 `1.0` 归档。后续驱动行为变更、协议能力变更或二进制不兼容时，必须显式提升清单中的版本。

## 1.1 工厂包 profile

Win 端配置工具负责现场交付、直连测试、配置发布和驱动包选择；平台端只维护基础数据、监测和配置，不管理驱动二进制。

工厂包支持三种 profile：

- `base`：只包含 `SystemMonitor`、`MqttDriver` 和 `pointctl`，适合批量初始化设备后再按项目 OTA 下发驱动。
- `project`：包含基础程序和 `edge-package-manifest.json` 中列出的项目驱动，是推荐的现场交付模式。
- `full`：包含当前所有驱动和工具，用于兼容旧流程或综合测试。

打包示例：

```bash
sh deploy/build-factory-package.sh --profile project --manifest /tmp/edge-package-manifest.json --out /tmp/gateway-factory-project.tar.gz
sh deploy/build-factory-package.sh --profile base --out /tmp/gateway-factory-base.tar.gz
sh deploy/build-factory-package.sh --profile full --out /tmp/gateway-factory-full.tar.gz
```

初始化示例：

```bash
sudo sh deploy/production-init.sh \
  --package /tmp/gateway-factory-project.tar.gz \
  --package-profile project \
  --machine-code COMM202600102
sudo sh deploy/install-factory-config.sh --profile project --manifest /tmp/edge-package-manifest.json
```

`machineCode` 是生产初始化的明确输入项，必须与平台登记的网关编号一致。初始化会写入 `config/runtime/device_identity.json`，并同步作为 MQTT `clientId`，后续配置拉取、实时监控、OTA、控制通道都按这个编号定向。

没有清单时，安装脚本默认按 `full` 执行，保持旧包兼容；包内存在 `edge-package-manifest.json` 时，默认按 `project` 执行。

## 2. 运行参数

程序支持：

```bash
./ModbusRtu --config /opt/modbus-gateway/config/runtime/devices/device_slave_ttySP1.json
```

开发调试时可使用：

```bash
./ModbusRtu --config config/runtime/devices/device_slave_ttySP1.json --mock
```

说明：

- Ubuntu 边端默认应走真实串口
- `--mock` 仅用于本地联调

## 3. 依赖

建议安装：

```bash
sudo apt-get update
sudo apt-get install -y libsqlite3-0
```

如果串口设备权限不足，建议把服务用户加入 `dialout`：

```bash
sudo usermod -aG dialout <user>
```

## 4. systemd

- `deploy/modbus-rtu@.service`
- `deploy/dlt645-driver@.service`
- `deploy/dio-driver@.service`
- `deploy/mqtt-driver@.service`
- `deploy/event-engine@.service`
- `deploy/compute-engine@.service`
- `deploy/local-display@.service`
- `deploy/local-kiosk@.service`
- `deploy/camera-service@.service`
- `deploy/system-monitor@.service`
- `deploy/mqtt-tls-tunnel@.service`
- `deploy/gateway-services.service`

OTA 脚本：

- `deploy/ota-apply.sh`
- `deploy/ota-rollback.sh`

安装方式示例：

```bash
sudo cp deploy/modbus-rtu@.service /etc/systemd/system/modbus-rtu@.service
sudo cp deploy/dlt645-driver@.service /etc/systemd/system/dlt645-driver@.service
sudo cp deploy/dio-driver@.service /etc/systemd/system/dio-driver@.service
sudo cp deploy/mqtt-driver@.service /etc/systemd/system/mqtt-driver@.service
sudo cp deploy/event-engine@.service /etc/systemd/system/event-engine@.service
sudo cp deploy/compute-engine@.service /etc/systemd/system/compute-engine@.service
sudo cp deploy/local-display@.service /etc/systemd/system/local-display@.service
sudo cp deploy/local-kiosk@.service /etc/systemd/system/local-kiosk@.service
sudo cp deploy/camera-service@.service /etc/systemd/system/camera-service@.service
sudo cp deploy/system-monitor@.service /etc/systemd/system/system-monitor@.service
sudo cp deploy/mqtt-tls-tunnel@.service /etc/systemd/system/mqtt-tls-tunnel@.service
sudo cp deploy/gateway-services.service /etc/systemd/system/gateway-services.service
sudo mkdir -p /etc/systemd/system.conf.d
sudo cp deploy/10-gateway-watchdog.conf /etc/systemd/system.conf.d/10-gateway-watchdog.conf
sudo cp deploy/gateway-services.sh /opt/modbus-gateway/bin/gateway-services.sh
sudo cp deploy/gateway-run.sh /opt/modbus-gateway/bin/gateway-run.sh
sudo cp deploy/production-smoke-test.sh /opt/modbus-gateway/bin/production-smoke-test.sh
sudo cp deploy/ota-apply.sh /opt/modbus-gateway/bin/ota-apply.sh
sudo cp deploy/ota-rollback.sh /opt/modbus-gateway/bin/ota-rollback.sh
sudo chmod +x /opt/modbus-gateway/bin/gateway-services.sh /opt/modbus-gateway/bin/gateway-run.sh /opt/modbus-gateway/bin/production-smoke-test.sh /opt/modbus-gateway/bin/ota-apply.sh /opt/modbus-gateway/bin/ota-rollback.sh
sudo systemctl daemon-reload
```

硬件看门狗建议由 systemd 统一管理：

- 业务进程退出、崩溃或被 kill 后，由各服务模板里的 `Restart=always` 自动拉起。
- 整机卡死、PID1 无法继续喂狗时，由 `/etc/systemd/system.conf.d/10-gateway-watchdog.conf` 配置的硬件 watchdog 触发设备重启。
- 修改 `RuntimeWatchdogSec` 后需要执行 `sudo systemctl daemon-reexec` 或重启设备，PID1 才会重新读取 Manager 配置；只执行 `daemon-reload` 不够。

如果之前已经单独 enable 过驱动实例，先停止并交给 `gateway-services.service` 管理：

```bash
sudo systemctl stop \
  'modbus-rtu@*.service' 'dlt645-driver@*.service' 'dio-driver@*.service' \
  'can-driver@*.service' 'iec-driver@*.service' 'compute-engine@*.service' \
  'event-engine@*.service' 'local-display@*.service' \
  'local-display-qt@*.service' 'local-kiosk@*.service' \
  'system-monitor@*.service' 'camera-service@*.service' \
  'mqtt-tls-tunnel@*.service' 'mqtt-driver@*.service' || true
sudo systemctl disable \
  'modbus-rtu@*.service' 'dlt645-driver@*.service' 'dio-driver@*.service' \
  'can-driver@*.service' 'iec-driver@*.service' 'compute-engine@*.service' \
  'event-engine@*.service' 'local-display@*.service' \
  'local-display-qt@*.service' 'local-kiosk@*.service' \
  'system-monitor@*.service' 'camera-service@*.service' \
  'mqtt-tls-tunnel@*.service' 'mqtt-driver@*.service' || true
```

开机自启动：

```bash
sudo systemctl enable gateway-services.service
sudo systemctl start gateway-services.service
```

`gateway-services.service` 会读取 app 配置中的 `deviceConfigFiles[]` 决定启动哪些协议驱动：

- 被 `deviceConfigFiles[]` 引用且 `protocol.type=modbus_rtu` 或 `modbus_tcp`：启动 `modbus-rtu@<设备配置文件名不含.json>`
- 被 `deviceConfigFiles[]` 引用且 `protocol.type=dlt645_2007`：启动 `dlt645-driver@<设备配置文件名不含.json>`
- 被 `deviceConfigFiles[]` 引用且 `protocol.type=local_dio`：启动 `dio-driver@<设备配置文件名不含.json>`
- 被 `deviceConfigFiles[]` 引用且 `protocol.type=can_socketcan` 或 `can`：启动 `can-driver@<设备配置文件名不含.json>`
- 被 `deviceConfigFiles[]` 引用且 `protocol.type=iec104`、`iec101`、`iec103`、`iec103_tcp` 或 `iec103_serial`：启动 `iec-driver@<设备配置文件名不含.json>`
- 如果 `mqtt-service.json:mqtt.enabled=true` 且数据上传、事件 outbox、OTA、实时请求或控制请求任一通道启用：启动 `mqtt-driver@mqtt-service`
- 如果 `mqtt-service.json:eventEngine.enabled=true`：启动 `event-engine@mqtt-service`
- 如果 `mqtt-service.json:computeEngine.enabled=true`：启动 `compute-engine@mqtt-service`
- 如果 `monitor-service.json:systemMonitor.enabled=true`：启动 `system-monitor@monitor-service`
- 如果 `monitor-service.json:localDisplay.enabled=true`：启动 `local-display@monitor-service`
- 如果 `monitor-service.json:localDisplay.kiosk.enabled=true` 且检测到显示输出：启动 `local-kiosk@monitor-service`
- 如果 `camera-service.json:cameraService.enabled=true` 且存在启用摄像头：启动 `camera-service@camera-service`
- 未被 `deviceConfigFiles[]` 引用的设备 JSON 不会启动

手动重扫配置：

```bash
sudo systemctl restart gateway-services.service
```

查看日志：

```bash
sudo journalctl -u gateway-services.service -f
sudo journalctl -u modbus-rtu@device_slave_ttySP1 -f
sudo journalctl -u dlt645-driver@device_dlt645_multi_meter_1_2 -f
sudo journalctl -u mqtt-driver@mqtt-service -f
```

生产健康检查：

```bash
sudo /opt/modbus-gateway/bin/gateway-run.sh list
sudo /opt/modbus-gateway/bin/gateway-run.sh health
sudo /opt/modbus-gateway/bin/gateway-run.sh stats
```

清理压测或异常退出留下的共享内存和临时目录：

```bash
sudo /opt/modbus-gateway/bin/gateway-run.sh cleanup
sudo /opt/modbus-gateway/bin/gateway-run.sh start
```

生产冒烟检查：

```bash
sudo /opt/modbus-gateway/bin/gateway-run.sh smoke
```

脚本会检查身份配置、运行文件、设备配置引用、TLS、systemd 服务、共享内存、磁盘、配置权限和 OTA 脚本。结果中 `fail=0` 才允许进入生产。

## 5. MQTT TLS

内置 MQTT Publisher 已支持 TLS。跨公网生产部署时，推荐直接在 app 配置启用：

```json
"broker": "ssl://broker.example.com:8883",
"tls": {
  "enabled": true,
  "caFile": "/opt/modbus-gateway/config/runtime/tls/ca.crt",
  "certFile": "",
  "keyFile": "",
  "insecureSkipVerify": false
}
```

边端必须具备 OpenSSL 运行库 `libssl` / `libcrypto`。`certFile` 和 `keyFile` 只在 broker 要求客户端证书时填写，且必须成对配置。

检查命令：

```bash
ldconfig -p | grep -E 'libssl|libcrypto'
```

如果使用 CA 文件校验服务端证书，先放入：

```text
/opt/modbus-gateway/config/runtime/tls/ca.crt
```

生产初始化可以直接完成证书部署。推荐流程是边端本机生成私钥和 CSR，平台 enrollment 接口用平台根 CA 签发客户端证书，边端把平台返回的 `ca.crt` 和 `<machineCode>-client.pem` 落盘：

```bash
sudo GATEWAY_HOME=/opt/modbus-gateway sh deploy/production-init.sh \
  --package /tmp/gateway-factory-project.tar.gz \
  --package-profile project \
  --machine-code COMM202600102 \
  --mqtt-broker ssl://kygate.kyxn.net:8883 \
  --mqtt-tls \
  --tls-generate-root-ca \
  --tls-platform-url https://edge.kyxn.net \
  --tls-token "$TLS_ENROLLMENT_TOKEN" \
  --tls-validity-days 365
```

`--tls-generate-root-ca` 会在 enrollment 前生成本地 bootstrap 根 CA 文件，便于现场离线检查证书目录；平台签发成功后，最终 `ca.crt` 以平台返回的 CA 为准。若 broker 启用 TLS，必须同时提供平台 URL 和签发 token，避免写出引用证书路径但没有客户端证书的半成品配置。

如果目标系统无法提供 OpenSSL 运行库，可启用本机 `stunnel` 兜底：

```bash
sudo apt-get install -y stunnel4
sudo cp config/runtime/tls/mqtt-service-stunnel.conf /opt/modbus-gateway/config/runtime/tls/mqtt-service-stunnel.conf
sudo vi /opt/modbus-gateway/config/runtime/tls/mqtt-service-stunnel.conf
```

stunnel 方案下 app 配置中的 `mqtt.broker` 改为：

```json
"broker": "tcp://127.0.0.1:18883"
```

`gateway-services.service` 会在 app broker 指向本机 stunnel 监听端口、且 `config/runtime/tls/*-stunnel.conf` 存在时自动启动 `mqtt-tls-tunnel@*.service`，无需单独 enable 隧道实例。

## 6. 多串口后台启动

多串口统一由 `gateway-services.service` 扫描配置后启动。不要再固定 enable 所有 `modbus-rtu@*` / `dlt645-driver@*` / `mqtt-driver@*` 实例，否则配置删除后旧实例仍可能开机启动。

## 7. 配置建议

`memoryStore` 推荐至少配置：

```json
{
  "sharedMemoryName": "gateway_point_store",
  "sqlitePath": "/opt/modbus-gateway/data/point_samples.db",
  "persistFlushIntervalMs": 60000,
  "writebackIntervalMs": 500,
  "writebackBatchSize": 100
}
```

串口参数放在：

```json
{
  "protocol": {
    "transport": {
      "serialPort": "/dev/ttyUSB0",
      "baudRate": 9600,
      "dataBits": 8,
      "stopBits": 1,
      "parity": "N",
      "timeoutMs": 1000
    }
  }
}
```
